## 새 파일

### `tools/ebay-sold-browser-audit.mjs`

eBay Sold/Completed 페이지를 브라우저로 열어서 일본어판 PSA10 거래완료 가격을 검수하는 스크립트.

주의:

- 이 스크립트는 일반 Node CLI 단독 실행용이 아님.
- Codex 브라우저 세션에서 `browser` global이 있을 때 실행됨.
- eBay 공식 Marketplace Insights Sold API는 현재 키로 403임.
- 그래서 공개 eBay Sold 검색 화면을 브라우저로 읽는 방식.

실행 형태 예시:

```js
const mod = await import('file:///C:/Users/kimtt/Documents/Codex/2026-06-21/https-youtu-be-rhuyy9lp72m-si-a4jhiygdebzzpvjf/tools/ebay-sold-browser-audit.mjs?run=1');
await mod.runSoldAudit({ start: 0, batchSize: 60, reset: true });
await mod.runSoldAudit({ start: 60, batchSize: 60, reset: false });
await mod.runSoldAudit({ start: 120, batchSize: 60, reset: false });
await mod.runSoldAudit({ start: 180, batchSize: 60, reset: false });
```

## 다음 작업 우선순위

1. `PSA10 eBay` 라벨을 더 명확히 바꾸기
   - 추천: `일본어판 PSA10 Sold`
   - 현재는 `일본어판 PSA10 eBay` + 아래 `eBay Sold · 표본 N건`

2. `High / Middle / Low` 의미 설명 추가
   - High: 최근 Sold 표본 85퍼센타일
   - Middle: 중앙값
   - Low: 15퍼센타일
   - 투자 판단용 가격 아님, 참고용 거래완료 분포임

3. eBay Sold 보조 소스 추가
   - 130point 또는 Terapeak 접근 가능하면 붙이기
   - 현재는 eBay Sold 화면 기반이라 eBay UI/언어 변경에 약함

4. 검수 스크립트 안정화
   - 지금은 Codex 브라우저 전용
   - 장기적으로 Playwright 설치형/CI형으로 바꾸면 좋음
   - 단, eBay 403/봇 차단 가능성 때문에 브라우저 세션 유지 방식이 현실적

5. 모바일 UI 추가 확인
   - 사용자가 실제 폰으로 보는 화면 기준으로 카드 가격 줄 간격 조정 가능
